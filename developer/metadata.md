# Work with metadata

Metadata describes apps, modules, tables, fields, enums, forms, menus, privileges, duties, and roles. Names are stable identifiers; labels are user-facing text.

Prefer the CLI or Web Designer over handwritten repetitive files. Schema synchronization is additive: adding tables, fields, and indexes is supported, while destructive changes need an explicit migration and backup plan.

Validate relationships, enum values, menu targets, and security artifacts together. Run the application and exercise generated list and form pages after every change.
